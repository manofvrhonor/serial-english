# 📘 Спецификация проекта «Serial English» (заморожено)

Полное ТЗ. Читать при **архитектурных изменениях** или когда нужны детали SRS / импорта / модели данных.
Для обычной сессии достаточно `agent-plan.md`.

---

## 1. Цель проекта

Веб-приложение для изучения английских слов и выражений из СЕРИАЛОВ и КНИГ.
Пользователь загружает субтитры (.srt) или текст книги (.txt по главам),
приложение извлекает незнакомые слова и устойчивые выражения и тренирует их
по карточкам с интервальными повторениями. Фокус — подготовка к конкретной
СЕРИИ или ГЛАВЕ перед просмотром/чтением.

- Перевод тренируется в ОБЕ стороны: English → Russian и Russian → English.
- Язык интерфейса: РУССКИЙ.

---

## 2. Принципиальные решения (зафиксировано, не пересматриваем)

- Никакой LLM / внешних API. Всё работает оффлайн, локально.
- Лемматизация — НЕ обрезка суффиксов. Индекс форм `forms.json.gz` (словоформа → лемма),
  строится при сборке словаря из лемм + неправильных форм. В рантайме — только lookup.
  Сокращения (`it's`, `let's`, `there's`…) раскрываются в токенизацию.
- Переводы — оффлайн словарная база en→ru (используется и для обратного направления).
- Выражения — оффлайн готовая база + ручное добавление. Что не поймалось — не беда.
- Клиентское веб-приложение, без бэкенда. Прицел на PWA позже.
- Хранение: IndexedDB (основное) + ручной экспорт/импорт .json. Облако — позже.

---

## 3. Стек

Vanilla JS (ES modules), SPA без сборщика, IndexedDB, CSS.
Оффлайн-данные: `dictionary.json.gz` + `forms.json.gz` (сжатые JSON).
Сборка словаря: `node scripts/build-dictionary.mjs` (нужен интернет один раз).
Локальный сервер для разработки: `python -m http.server 8081`.

---

## 4. Разделы интерфейса

Боковое меню + рабочая область:

1. Импорт
2. **База знаний** — хаб с переключателем **Слова / Выражения**, статистикой по типу, секциями **На изучении**, **Выучено**, **Стоп-лист**; поиск + модал «Добавить»
3. Тренировка
4. Сериалы (вложенность: Сериал → Сезон → Серия)
5. Книги (вложенность: Книга → Глава)
6. Настройки (интервалы SRS, экспорт/импорт .json, HARD RESET)

**HARD RESET:** два модальных шага — предупреждение со списком удаляемого → финальный
вопрос «ВЫ ТОЧНО ХОТИТЕ ВСЁ УДАЛИТЬ?». Кнопки столбцом на всю ширину; отмена (Отмена/Нет)
— тёмные (`btn-dark`), подтверждение — красные. Сброс: `hardResetState()` в `database.js`.

Отдельные пункты «Слова» и «Выражения» в меню **убраны** — редактирование карточек
в **База знаний** (переключатель Слова/Выражения + секция **На изучении**).
Стоп-лист и статистика — в **База знаний**, не в Настройках.

**Добавление вручную:** строка поиска; если по запросу нет совпадений — кнопка «Добавить»
→ модал `#kb-add-modal` (слово/выражение + до 3 переводов-чипов). Доступно в «На изучении»,
«Выучено» и «Стоп-лист» (только слова).

## 5. Поток обработки импорта

Поддерживаются два типа источника:

- .srt → серия (внутри сезона, внутри сериала);
- .txt → глава (внутри книги).

Конвейер:
файл → **автопарсинг и разбор** → лемматизация → исключение: стоп-лист пользователя, База знаний (выучено),
слова «на изучении» → поиск выражений по базе
→ подстановка переводов из словарной базы
→ **шаг 1:** список/карточки — только **новые слова/фразы с переводом**
→ **шаг 2:** после обработки шага 1 — **без перевода** (вписать вручную или стоп-лист)
→ попап «Добавить выбранные в словарь» (список + ОК / Поправить)
→ после ОК: частичный коммит; при завершении обоих шагов — итог на экране; мета-форма сбрасывается

На шаге 1 показывается **статистика по файлу** (всего / знаю / стоп / на изучении / новых)
и две строки: «Слова с переводом — N» / «Без перевода — M».
На шаге 2 — текст «Нашлось N слов без перевода…» без общей сводки.

Счётчик **«новых»** в статистике — по статусу (`!known && !stop && !studying`), **не** по галочке выбора.

В списке: **«Выбрать всё»** и **«Выбрано: N»** (N — отмеченные среди видимых на текущем шаге).

Экран подтверждения показывает:

- новые слова (с переводами, чекбоксы);
- новые выражения (чекбоксы);
- слова БЕЗ перевода — пометка «нет перевода»: вписать вручную или исключить;
- пропущенные (уже известные) — счётчиком.

Кнопка загрузки: **«Загрузить новый файл»** (единственная точка входа для нового файла).

**Импорт из библиотеки:** кнопка «Выбрать из библиотеки» → готовые наборы из `data/library/` (см. §5.1). Нужен локальный HTTP-сервер или GitHub Pages; `file://` не поддерживается.

При импорте пользователь указывает, КУДА грузить:

- для .srt: Сериал → Сезон → Серия;
- для .txt: Книга → Глава.

Метки берутся из имени файла; при неудаче — задаются вручную.

Каждое слово помнит СПИСОК ИСТОЧНИКОВ (связь многие-ко-многим) — это могут
быть и серии, и главы книг одновременно.

## 5.1. Библиотека сериалов (`data/library/`)

Статические JSON в репозитории; коммитятся после авторинга в админ-режиме.

- `index.json`: `{ "shows": [ { "id", "title", "file", "seasons", "episodes" } ] }`
- `<id>.json`: `{ "id", "title", "seasons": [ { "number", "episodes": [ { "number", "title", "words": ["lemma"], "phrases": ["text"] } ] } ], "translations": { "words": { "lemma": ["перевод"] }, "phrases": { "text": [...] } } }`

Загрузчик: `js/import/library.js` (`getLibraryIndex`, `getLibraryShow`, `importLibraryEpisodes`). Переводы из файла библиотеки имеют приоритет над общим словарём; known/stop пропускаются.

Админ: Настройки → админ-режим (пароль `54321`) → «Админ-библиотека» — zip субтитров, курирование, экспорт show/index.

## 6. Структура «Сериалы» и «Книги» + готовность

Сериалы: дерево Сериал → Сезон → Серия. В шапке карточки сериала — плашка
**N/M изучено** (агрегат по всем сериям). У каждой серии — полоска прогресса
**без текста** и кнопка «Подготовка к просмотру». Без подзаголовка «Готовность = …»
и без дублирования N/M у прогресс-бара сериала и в строке серии.

Книги: дерево Книга → Глава. **Аналогично** сериалам: одна плашка на книгу,
полоска у главы без цифр, кнопка «Подготовка к чтению».

Слово, ушедшее в Базу знаний, повышает готовность всех источников, где оно
встречается (серий и глав).

## 7. Разделы «Слова» и «Выражения» (UI в «База знаний»)

Функционально как в MVP; в интерфейсе — **База знаний** с переключателем **Слова / Выражения**
и секцией **На изучении** (таблицы в `study-words.js` / `study-phrases.js`).
Структура идентична. Поля элемента:

- слово (english) / выражение — в данных `lemma` / `text`; в UI колонка **«Слово»**, не «Лемма»;
- переводы (топ-3);
- ДВА блока SRS — EN→RU и RU→EN (уровень, галочки, дата показа у каждого);
- список источников (серии / главы); для слов **добавленных вручную** (без источника) —
  кнопка ≡ → «Добавлено вручную»;
- статус «выучено».

Действия: редактировать переводы и их порядок (топ-3), удалить, пометить
выученным (→ База знаний), добавить вручную (поиск → «Добавить» → модал), поиск/фильтр.

**Иконки действий** (единый набор `js/ui/action-icons.js`):
✓ выучено · ✕ удалить/убрать · 🚫 в стоп-лист · ↩ вернуть в изучение · ≡ источники.
Одинаковый вид в «На изучении», «Выучено», «Стоп-лист» и в импорте (список).

Статистика в шапке: **в работе** — все не выученные элементы; **к повторению сегодня** —
число карточек с наступившим SRS (`countTrainingItems`, направления EN→RU и RU→EN
считаются как один элемент). Отдельно: `noTransWords` / `noTransPhrases` — без перевода
на вкладке «На изучении» (фильтр «Без перевода»).

Переводы (топ-3): при добавлении пользователь видит полный список значений
из словарной базы и выбирает ДО 3, расставляя приоритет (топ-1, топ-2, топ-3).

**Отображение в UI:** `titleCase()` в `js/core/display-text.js` — каждое слово с
заглавной буквы (EN/RU, карточки, чипы переводов, списки). Хранение в IndexedDB
без нормализации регистра; поиск — case-insensitive.

## 8. Тренировка и SRS

### 8.1 Режимы предъявления

Подписи и подсказки — `TRAINING_MODE_LABELS` / `TRAINING_MODE_HINTS` в `js/core/srs.js`
(setup тренировки и badge на карточке используют один набор).

1. Слово + перевод (показано сразу) → «Знал / Не знал».
2. Перевод по клику (скрыт, раскрывается) → «Знал / Не знал».
3. 4 варианта (выбор).
4. МИКС — каждая карточка случайно в одном из трёх режимов.

### 8.2 Направление перевода

- EN→RU: показывается английское слово, подбирается русский перевод.
- RU→EN: показывается русское слово, подбирается английское.
- Оба — случайно чередуются.

Режим «4 варианта»: среди 4 вариантов ТОЛЬКО ОДИН правильный. Правильный из
топ-1/2/3 с весами; 3 неправильных — по одному из **других** элементов базы
(без заглушек «вариант N»). Если дистракторов < 3 — режим недоступен в setup;
в сессии fallback на «Перевод по клику». Очередь: Fisher-Yates + без соседних
карточек одного слова при «Оба направления» (`shuffleAvoidingAdjacent`).

### 8.3 SRS — два независимых направления

У каждого слова ДВА блока SRS: `enru` и `ruen`. У каждого:

- уровень 0 → 5, интервалы [1, 3, 7, 16, 30] дней;
- 3 галочки по режимам (mode1/2/3);
- своя дата `due`.

ВЫУЧЕНО: когда ОБА направления на уровне 5 с собранными галочками.

### 8.4 Старт тренировки

Выбор: Слова / Выражения / Всё, направление, режим, «только к повторению сегодня».
Вход из Сериалов/Книг — подготовка к серии/главе.

**Шаги:** 1 шаг = 10 карточек (`STEP_SIZE` в `training.js`). Статистика «Сегодня» —
в разделе **Тренировка**, не в Настройках.

**Mobile-сессия:** заголовок «Тренировка» скрыт (активный пункт nav достаточен);
badges режим/прогресс/направление остаются. В режиме «4 варианта» обратная связь
«Верно!» / «Не verno!» — **на карточке под словом**, не под списком вариантов.

## 9. Модель данных (json v2)

Реализация и миграции: `js/db/database.js`. Формат экспорта/импорта.

Ключевые сущности: `settings` (intervals, stopList, sessionHistory), `shows`, `books`,
`words`, `phrases`, `knowledge`. SRS: `{ level, checks: {mode1,mode2,mode3}, due }` на
каждое направление (`enru`, `ruen`). `stopList`: `[{ lemma, translations }]`.

Полный пример схемы — в git-истории `agent-plan.md` (до рефакторинга 2026-06) или
в экспортированном .json пользователя.

## 10. Что НЕ входит в MVP (отложено)

- Облачная синхронизация (только ручной экспорт/импорт .json).
- Полный SM-2 (используем минимальный SRS).
- PWA-обёртка (архитектурно готовим, оформляем позже).
- Любой LLM / API — не планируется вообще.

## 11. Поэтапный план реализации

**Все этапы 1–9 выполнены** (MVP + UI). Исторический план:

1. Каркас — стек, навигация, IndexedDB, экспорт/импорт.
2. Модель данных — json-схема v2, shows/books, двойной SRS.
3. Импорт — .srt/.txt, лемматизация, словарь, подтверждение.
4. Слова / Выражения — списки, топ-3, выучено.
5. Тренировка + SRS — 3 режима + МИКС, шаги по 10.
6. Сериалы и Книги — деревья, готовность, подготовка.
7. База знаний — выученное, стоп-лист, исключение из импорта.
8. Полировка — настройки, HARD RESET, UX.
9. UI / дизайн — Lovable-референс, mobile + desktop.
