## 2. Архитектура и решения (ПОЧЕМУ так)

> При правке UI не возвращать «как в Lovable» или «как проще» — сверяться с обоснованием.

### 2.1 Информационная архитектура

| Решение | Почему |
|---------|--------|
| Единый хаб **База знаний** (не отдельные «Слова»/«Фразы» в меню) | Меньше дублирования; стоп-лист и статистика рядом с карточками |
| Стоп-лист и блок «База» в **База знаний**, не в Настройках | Настройки = только системное: интервалы, backup, HARD RESET |
| **Сериалы** и **Книги** — отдельные пункты меню | Фокус продукта — подготовка к серии/главе; отличие от Lovable |
| В UI **«Слово»**, в данных **`lemma`** | Понятнее пользователю; поле в JSON не переименовывать |
| Настройки: только интервалы, экспорт/импорт, HARD RESET | Всё остальное — рабочие разделы, не конфиг |

### 2.2 Shell: desktop и mobile

| Решение | Почему |
|---------|--------|
| Desktop: **sidebar** слева; toggle в **низу** sidebar (`.sidebar-footer`) | Контент на всю высоту; header убран — занимал ~3rem без пользы |
| Нет `desktop-header`, нет «Оффлайн словарь» | Лишняя строка на каждой вкладке |
| Mobile: **6 кнопок** в `mobile-nav` (Импорт·База·Тренировка·Сериалы·Книги·Настройки) | Прямой доступ; sheet «Ещё» добавлял лишний клик |
| Нет `mobile-header` | Дублировал активный пункт нижнего меню |
| `h1.view-title-section` **скрыт на mobile** | Заголовок = пункт nav; остаётся подзаголовок-описание |
| Скролл в **`.content`**, не `window` | `html,body { overflow:hidden }`, `.app { height:100dvh }` — иначе ломается layout и scroll-top |
| Дизайн: oklch-токены, **card-layout** | Референс Lovable; не откатывать к «голому» HTML |

### 2.3 Списки и таблицы (База знаний)

| Решение | Почему |
|---------|--------|
| **Слова / Выражения** — переключатель **сверху** (не только в «На изучении») | Статистика и списки всегда про один тип; меньше путаницы |
| «К повторению сегодня» = `countTrainingItems(…, dueOnly: true)` по типу | Один элемент, не сумма EN→RU и RU→EN |
| «В работе» = все не выученные на вкладке «На изучении» | Включая слова без перевода |
| Фильтр **«Без перевода (N)»** на «На изучении» при N > 0 | Счётчик `noTransWords` / `noTransPhrases`; дополнить переводы вручную |
| Поиск в Выучено/Стоп-лист: обновлять только `#kb-content` | Иначе теряется фокус в `#k-search` |
| **Один** поиск `#k-search`; кнопка **«Добавить»** только если запрос не пуст и **0 совпадений** в текущей вкладке | Нет дублирующих inline-форм; добавление через модал |
| Добавление: модал `#kb-add-modal` — слово/выражение + чипы перевода + Сохранить | Словарь подставляет переводы; секции: На изучении / Выучено / Стоп-лист (слова) |
| Стоп-лист на вкладке **Выражения** — пустое сообщение | В модели данных стоп-лист только для слов |
| SRS **не показывается** в таблице «На изучении» | SRS — внутренняя механика; данные в `word.srs` / `phrase.srs` |
| Источники в **попапе** (`sources-modal`), не колонкой | Экономия места на mobile; краткий формат в кнопке: `formatSourceShort()` |
| Слово **вручную** → кнопка ≡ активна; в попапе **«Добавлено вручную»** | `word.manual` или пустой `sources`; см. `sources-modal.js` |
| Колонки: **Слово \| Переводы \| 3 кнопки** (✓ выучено · ✕ удалить · ≡ источники) | Фиксированная таблица; кнопки — колонка на mobile, ряд на desktop |
| **Единые SVG-иконки** — `js/ui/action-icons.js` | ✓ выучено · ✕ удалить/убрать · 🚫 стоп-лист · ↩ вернуть · ≡ источники — одинаково в «На изучении», «Выучено», «Стоп-лист», импорте |
| Без тегов **изучаю / выучено / стоп** в строках | Вкладка уже задаёт контекст |
| Оставлены теги **в карточках**, **ручное** (выражения) | Информативны, не дублируют вкладку |

### 2.4 Импорт

| Решение | Почему |
|---------|--------|
| Кнопка **«Загрузить новый файл»** по центру; hint **«srt или txt»** | Один вход для нового файла; без второй кнопки «другой файл» после коммита |
| **Авторазбор** сразу после загрузки | Без превью текста и без шага «Разобрать» |
| Метаданные .srt: **2 строки** — (Сериал+Сезон), (Название серии+Серия) | Компактнее на mobile |
| **Два шага** по словам: сначала **с переводом**, затем **без перевода** | Сначала быстрый импорт; без перевода — вручную или в стоп |
| Табы **Слова / Фразы** и **Список / Карточки** — на всю ширину; по умолчанию **Список** | Единый UX; список удобнее для длинных файлов |
| Кнопка **«Добавить выбранные в словарь»** — **над** табами | Не теряется при прокрутке длинного списка |
| В списке: **«Выбрать всё»** + **«Выбрано: N»** (только видимые на шаге) | Без фильтров «Новое/Без перевода» — шаг задаёт состав |
| Статистика «новых» в шапке — по **статусу** слова, не по галочке `included` | `analyzeSummary`: `!known && !stop && !studying && !removed` |
| На шаге «без перевода» — отдельный текст-подсказка, без общей сводки файла | Фокус на ручном вводе / стоп-листе |
| Попап подтверждения (**ОК / Поправить**) перед коммитом | Защита от случайного импорта |
| Сессия импорта: несколько коммитов → итог; мета-форма сбрасывается только в конце | Можно пройти оба шага одного файла без перезагрузки |
| Swipe-карточки + trans-chips; иконки ✓/🚫 в списке | Референс Lovable; те же иконки, что в Базе знаний |

### 2.5 Тренировка

| Решение | Почему |
|---------|--------|
| Без подсказки «1 шаг = 10 карточек…»; без заголовка «Последние шаги»; в истории **без колонки «Источник»** | Меньше шума; источник не нужен в сводке шагов |
| `STEP_SIZE = 10` | Короткие сессии; статистика шагов в **Тренировка**, не Настройки |
| Двойной SRS: **`enru`** и **`ruen`** независимо | EN→RU и RU→EN — разные навыки; см. `docs/agent-spec.md` §8 |
| «4 варианта»: промпт в карточке, варианты снаружи | Lovable-паттерн; не сворачивать внутрь карточки |
| Подготовка к серии/главе: `ctx.startPrepTraining()` | См. §4 — не импортировать router из views |

### 2.6 Данные и логика

| Решение | Почему |
|---------|--------|
| `stopList`: **`{ lemma, translations }`**, не строки | При удалении слова переводы сохраняются; миграция `normalizeStopList()` |
| Стоп-лист **пустой** при первом запуске | Пользователь сам наполняет; встроенный ~40 слов удалён |
| «Знаю» при импорте → карточка с переводами | Не только запись в `knowledge` |
| Лемматизация = **lookup** в `forms.json.gz` | Не обрезка суффиксов — см. spec §2 |
| IndexedDB + ручной .json | Без бэкенда; данные не синхронизируются между устройствами |

### 2.8 Сериалы и Книги

| Решение | Почему |
|---------|--------|
| Счётчик **N/M изучено** — только **плашка** в шапке карточки сериала/книги | Не дублировать цифры у прогресс-бара и у каждой серии/главы |
| Без подзаголовка «Готовность = …» под заголовком страницы | Цифра уже в плашке; меньше шума на mobile |
| У серии/главы — **полоска** прогресса без текста + кнопка «Подготовка» | Визуальный прогресс без повторения N/M |

### 2.9 Отличия от Lovable (намеренные)

- Единый **База знаний** вместо отдельных Словарь / Фразы / Известные.
- **Сериалы** и **Книги** — отдельные пункты меню (не внутри одного раздела).

## 3. Ограничения (не нарушать)

- Без LLM / внешних API — всё оффлайн.

- Язык UI: **русский**.
- Отложено: PWA, облако, полный SM-2 — см. `docs/agent-spec.md` §10.

## 7. Следующий шаг

### Отложено: прогресс серий/книг по словарю файла (P3)

**Проблема:** `calcReadiness` считает только карточки с `sources.includes(episodeId)`. Слова, уже известные до импорта нового файла, **не привязываются** к новому источнику → прогресс E02 может быть 0/N, хотя 60/200 уже «знаете».

**Целевая модель:**
- При разборе файла сохранять **снимок лексики** на эпизод/главу (`vocabulary: { words, phrases }`).
- Прогресс = сверка снимка с **глобальным** статусом: выучено · стоп · без перевода · в работе · новое.
- Многоцветный бар: красный (стоп) · оранжевый (без перевода) · зелёный (выучено); подпись «495/500 готово»; тултип с разбивкой.
- Сводка после разбора: «200 слов: 60 знаете · 5 стоп · 3 без перевода · 132 новых».
- Подготовка к просмотру — только «не готово» по снимку.

**Файлы:** `js/core/readiness.js`, `js/db/database.js` (схема ep/ch), `js/views/import.js`, `js/views/shows.js`, `js/views/books.js`.

**Открытые решения:** «знаю» = зелёный?; обновлять снимок при повторном импорте?; фразы — та же схема.

Прикреплять: `@agent-plan.md` + «реализуй P3 прогресс серий».
