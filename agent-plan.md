## 1. Цель проекта
Веб-приложение для изучения английских слов и выражений из СЕРИАЛОВ и КНИГ.
Пользователь загружает субтитры (.srt) или текст книги (.txt по главам),
приложение извлекает незнакомые слова и устойчивые выражения и тренирует их
по карточкам с интервальными повторениями. Фокус — подготовка к конкретной
СЕРИИ или ГЛАВЕ перед просмотром/чтением.

- Перевод тренируется в ОБЕ стороны: English → Russian и Russian → English.
- Язык интерфейса: РУССКИЙ.

---

## 2. Принципиальные решения (зафиксировано, не пересматриваем)
- Никакой LLM / внешних API. Всё работает оффлайн, локально.
- Лемматизация — НЕ обрезка суффиксов. Индекс форм `forms.json.gz` (словоформа → лемма),
  строится при сборке словаря из лемм + неправильных форм. В рантайме — только lookup.
  Сокращения (`it's`, `let's`, `there's`…) раскрываются в токенизацию.
- Переводы — оффлайн словарная база en→ru (используется и для обратного направления).
- Выражения — оффлайн готовая база + ручное добавление. Что не поймалось — не беда.
- Клиентское веб-приложение, без бэкенда. Прицел на PWA позже.
- Хранение: IndexedDB (основное) + ручной экспорт/импорт .json. Облако — позже.

## 3. Стек
Vanilla JS (ES modules), SPA без сборщика, IndexedDB, CSS.
Оффлайн-данные: `dictionary.json.gz` + `forms.json.gz` (сжатые JSON).
Сборка словаря: `node scripts/build-dictionary.mjs` (нужен интернет один раз).
Локальный сервер для разработки: `python -m http.server 8080`.

## 5. Поток обработки импорта
Поддерживаются два типа источника:
- .srt → серия (внутри сезона, внутри сериала);
- .txt → глава (внутри книги).

Конвейер:
файл → парсинг → лемматизация → исключение: стоп-лист пользователя, База знаний (выучено),
слова «на изучении» → поиск выражений по базе
→ подстановка переводов из словарной базы
→ экран списка (редактирование) — **только новые** слова/фразы
→ попап «Импортировать выбранные» (список + ОК / Поправить)
→ после ОК: импорт в базу, список и форма файла сбрасываются, итог на экране

После «Разобрать» показывается **статистика по файлу**: всего уникальных слов,
сколько знаете / в стоп-листе / на изучении / новых. В списке — только новые.

Экран подтверждения показывает:
- новые слова (с переводами, чекбоксы);
- новые выражения (чекбоксы);
- слова БЕЗ перевода — пометка «нет перевода»: вписать вручную или исключить;
- пропущенные (уже известные) — счётчиком.

При импорте пользователь указывает, КУДА грузить:
- для .srt: Сериал → Сезон → Серия;
- для .txt: Книга → Глава.
Метки берутся из имени файла; при неудаче — задаются вручную.

Каждое слово помнит СПИСОК ИСТОЧНИКОВ (связь многие-ко-многим) — это могут
быть и серии, и главы книг одновременно.

## 6. Структура «Сериалы» и «Книги» + готовность
Сериалы: дерево Сериал → Сезон → Серия. У каждой серии — ПРОГРЕСС-БАР
готовности = выученные / все слова серии. Кнопка «Подготовка к просмотру» →
тренировка по неизученным словам серии. Агрегация готовности до сезона/сериала —
опционально (украшение).

Книги: дерево Книга → Глава. Полностью аналогично: загрузка .txt по главам,
словарь главы, прогресс-бар готовности, кнопка «Подготовка к чтению».

Слово, ушедшее в Базу знаний, повышает готовность всех источников, где оно
встречается (серий и глав).

---

## 7. Разделы «Слова» и «Выражения» (UI в «База знаний»)
Функционально как в MVP; в интерфейсе — **База знаний → На изучении** (Слова / Выражения).
Структура идентична. Поля элемента:
- слово (english) / выражение — в данных `lemma` / `text`; в UI колонка **«Слово»**, не «Лемма»;
- переводы (топ-3);
- ДВА блока SRS — EN→RU и RU→EN (уровень, галочки, дата показа у каждого);
- список источников (серии / главы);
- статус «выучено».

Действия: редактировать переводы и их порядок (топ-3), удалить, пометить
выученным (→ База знаний), добавить вручную, поиск/фильтр.

Переводы (топ-3): при добавлении пользователь видит полный список значений
из словарной базы и выбирает ДО 3, расставляя приоритет (топ-1, топ-2, топ-3).

## 8. Тренировка и SRS

### 8.1 Режимы предъявления
1. Слово + перевод (показано сразу) → «Знал / Не знал».
2. Перевод по клику (скрыт, раскрывается) → «Знал / Не знал».
3. 4 варианта (выбор).
4. МИКС — каждая карточка случайно в одном из трёх режимов.

### 8.2 Направление перевода
В тренировке выбирается направление:
- EN→RU: показывается английское слово, подбирается русский перевод.
- RU→EN: показывается русское слово, подбирается английское.
- Оба — случайно чередуются.

Режим «4 варианта» работает в обе стороны:
- EN→RU — варианты русские переводы;
- RU→EN — варианты английские слова.
Правило: среди 4 вариантов ТОЛЬКО ОДИН правильный. Правильный берётся из
топ-1/2/3 с весами: топ-1 чаще, топ-2 средне, топ-3 реже. 3 неправильных —
из переводов/слов других элементов базы.

### 8.3 SRS — два независимых направления
У каждого слова ДВА независимых блока SRS: enru и ruen. У каждого:
- уровень 0 → 5, интервалы [1, 3, 7, 16, 30] дней (уровень 0 = сегодня);
- 3 ГАЛОЧКИ по режимам (по одной на режим 1/2/3);
- своя дата следующего показа.

Логика (для каждого направления отдельно):
- верный ответ в режиме → ставит галочку этого режима В ДАННОМ направлении;
- собраны все 3 галочки направления → уровень этого направления +1, новый
  интервал, галочки обнуляются;
- ошибка → уровень этого направления -1 (не ниже 0; НА 0 ОСТАЁТСЯ 0),
  галочки направления сброшены, показ — скоро.

Слово может иметь, например, EN→RU на уровне 5, а RU→EN на уровне 1 — это
нормально и ожидаемо.

ВЫУЧЕНО (→ База знаний): когда ОБА направления достигли максимального
уровня (5) с собранными галочками.

В сессию берутся карточки/направления с датой показа ≤ сегодня. В МИКС режим
карточки = тот, в котором её показали (двигает соответствующую галочку).

### 8.4 Старт тренировки
Выбор: что тренируем (Слова / Выражения / Всё), направление (EN→RU / RU→EN / Оба),
режим (1/2/3/МИКС), «только к повторению сегодня» (вкл/выкл). Плюс отдельный вход
из «Сериалов» / «Книг» (подготовка к серии/главе).

**Шаги (реализовано):** 1 шаг = 10 карточек. После шага — короткая статистика,
кнопка «Следующий шаг →». Статистика «Сегодня» и история шагов — в разделе «Тренировка»,
не в «Настройках». Константа `STEP_SIZE = 10` в `js/views/training.js`.

### Словарь и формы (сессия 2025-06)
- Источники переводов при сборке: iuzhakov → spishniak; `translation-overrides.json` — финальный приоритет.

- Пересборка: `node scripts/build-dictionary.mjs`
- Плохие переводы править в `data/translation-overrides.json`, затем пересобрать.

### SRS-схема — выровнена (Этап 5)

### Тренировка: шаги (сессия 2025-06)
- 1 шаг = 10 карточек (`STEP_SIZE` в `training.js`).
- Очередь строится один раз при «Начать»; шаги — срезы очереди.
- `settings.sessionHistory` — одна запись на каждый завершённый шаг.
- `getTodayTrainingSummary()` — агрегация шагов за сегодня.

### HARD RESET (сессия 2025-06, обновлено 2026-06)
`hardResetState()` в `database.js` сбрасывает: words, phrases, shows, books,
knowledge, sessionHistory, **стоп-лист (очищается)**. **Не трогает:** офлайн-файлы data/*.json(.gz).
**Остаётся в IndexedDB:** дефолтные intervals [1,3,7,16,30]. Стоп-лист после reset — **пустой**
(пользователь наполняет сам; встроенного списка больше нет).
